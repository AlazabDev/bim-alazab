# BIM Operating Model

This document defines the first operating model for Alazab BIM.

## Scope

Alazab BIM manages project information, BIM files, approvals, issues, RFIs, submittals, technical evidence, handover records, and post-handover maintenance references.

## Core flow

Project -> Discipline -> Record -> Revision -> Status -> Decision

## Primary modules

- Projects
- Files and revisions
- Approvals
- Issues
- RFIs
- Submittals
- Technical evidence
- Handover
- Post-handover maintenance

## Decision rule

No technical decision should be accepted without a source record such as a drawing, document, photo, report, issue, approval, RFI, or audit entry.
